# .skill.md — Skill: repo-ops (run tests, lint, build docs)

Name: `repo-ops`

Purpose: Provide a compact skill for common repository operations: install deps, run tests, lint, and build docs. Agents can invoke this skill when they need to validate changes or produce artifacts.

Actions:

- `install`: `uv sync` — set up dev environment.
- `test`: `uv run pytest -q` or `python -m pytest tests/` — run unit tests; return failing tests and traces.
- `lint`: `ruff check .` — run linter (configured in `pyproject.toml`).
- `docs`: `make -C docs html` — build documentation (install docs deps first).

Inputs:

- `action`: one of `install|test|lint|docs`.
- `focus` (optional): path or module to scope operation (e.g., `tests/test_tabulate.py`).

Outputs:

- For `test`: a concise summary — `X passed, Y failed` and top 5 failures with tracebacks.
- For `lint`: number of issues and first 10 messages.
- For `docs`: path to built docs (`docs/_build/html/`) and any warnings.

Notes for implementers:

- Prefer using `uv` to reproduce development environment. If `uv` is unavailable, fallback to `pip install -e .`.
- Do not modify `pyproject.toml` version fields automatically.
- If failures are environment-specific (e.g., missing external tools), report the missing dependency and suggested remediation.
